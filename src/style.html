Cortex Style Guide

Introduction

The aesthetic style of Cortex is built on the official Sass port of Bootstrap. Bootstrap's built-in responsive layout system simplifies the task of building a site that is compatible and usable on multiple computing and mobile device platforms. To minimize the writing of redundant stylesheet code in our custom Cortex-specific stylesheet, native Bootstrap-defined styles are used whenever possible and style definitions are overwritten with Cortex-specific customizations as needed. There are instances when one-off style definitions may be needed to control the look and feel of a specific element. These definitions should be created sparingly and used only when no appropriate, semantic native Bootstrap-defined style is available.

Navigation

Site navigation within Cortex is defined at three levels:

  1. Header (Primary) Navigation
  2. Breadcrumb (Hierarchical) Navigation
  3. Left (Categorical) Navigation

Header navigation is present on all pages within the Cortex application following user authentication with the exception of transitional, modal page states. From here, a Cortex user can access all modules of the Cortex application. Also accessible from the header navigation is a link to the user's profile as well as the ability to log out of Cortex.

Breadcrumb navigation is also present on all pages within the Cortex application following user authentication. The breadcrumb trail of hyperlinks allows the user to see where in the Cortex site hierarchy they are currently located.

Left navigation is available on pages where additional functionality is available to manipulate the current page view to highlight or filter its contents down to a smaller subset of information. Any hyperlinks found in left navigation should link to either a child page or a child function of the current page view.

Color

The primary color scheme of Cortex is black text on a white background with a three-color gradient header spanning hex #022745, #104070, #09335B blues and decorative elements colored hex #f47937 orange.

Typography

The Cortex page header uses the PT Sans font with a base font size of 14px.

All body typography is declared as font-family: "Helvetica Neue", Helvetica, Arial, sans-serif with a base font size of 14px as defined by Bootstrap.

Headings

Cortex uses the default Bootstrap heading styles. The page heading at the top of each page is <h3>.

Forms

Forms in Cortex are structured using HTML5 description lists. Bootstrap provides a horizontal-flowing description list class dl-horizontal that allows a label to be placed side-by-side with the form field when the form label is placed within <dt> and the form field within <dd>. This horizontal description list is responsive-friendly and semanatically correct design for side-by-side label/field form layouts.

Preceding each group of form elements within a description list is a <legend> that serves as a heading to the grouped form elements. It has a style defined with an orange border below the text.

Buttons

Buttons in Cortex use the Bootstrap button style definitions. Each button uses the btn and btn-sm classes for sizing. Buttons are further styled using classes btn-default for standard buttons and btn-primary for primary highlighted buttons. Primary highlighted buttons are used to guide the user to a desired form action. When multiple choices are presented at the end of a form, such as in the case of Save and Cancel, the action for proceeding, assuming the form is correctly filled out, will be highlighted. In this case, Save is the primary highlighted button.

For btn-default, the Bootstrap default button border color has been replaced with color: #f47937. Primary buttons use the background mixin @include gradient-vertical(#FFA178, #E9743A, 0%, 100%)

Buttons also have defined styles for a disabled state as well as a :hover pseudoclass.


        <button class="btn btn-sm btn-primary">Primary Button</button>
        <button class="btn btn-sm btn-default">Default Button</button>
Button Icons and Labels

Button icons in Cortex buttons utilize the Font Awesome icon library. These icons may be placed to the left of a button label and match the button text color, or they may be placed by themselves in a non-ambiguous manner as an icon that represents the button's function. For instance, a button that consists of only the magnifying glass icon fa-search signifies a Search button.

Button labels should be used whenever possible, though it is okay to leave off button labels in lieu of only an icon given that:

  1. The button is used to supplement an adjacent form field AND
  2. The button is not used for navigation from one distinct page state to another.
